Claude Token Counter工具现已升级，新增模型对比功能。开发者Simon Willison对其Claude Token Counter工具进行了更新，现在用户可以针对同一段文本在不同Claude模型上运行token计数并进行比较。工具支持当前所有主要Claude模型，包括Opus 4.7、Opus 4.6、Sonnet 4.6和Haiku 4.5。，此次升级的背景是Anthropic发布的Claude Opus 4.7模型采用了更新的tokenizer。根据官方说明，新tokenizer改进了文本处理方式，但代价是相同输入可能映射到更多token，数量约为旧版本的1.0至1.35倍，具体取决于内容类型。实际测试发现，将Opus 4.7的系统提示词输入工具后，其token数量达到了Opus 4.6的1.46倍。尽管输入输出token定价与4.6版本相同（每百万输入token 5美元，每百万输出token 25美元），但由于token膨胀，实际使用成本预计将增加约40%。，工具同样支持图像token计数。Opus 4.7增强了图像处理能力，可接受最长边达2576像素（约375万像素）的高分辨率图像，处理能力是之前模型的三倍以上。测试显示，一张3456x2234像素的PNG图像在Opus 4.7上产生的token数量是4.6的3.01倍。但后续测试表明，这种大幅增加主要源于高分辨率支持：对于一张682x318像素的小图，两个模型消耗的token数量几乎相同（4.7为314，4.6为310）。，此外，针对文本密集型内容的测试显示，一个15MB、30页的PDF文件在Opus 4.7上产生60,934个token，而在4.6上为56,482个，倍率仅为1.08倍，远低于纯文本的膨胀比例。这表明tokenizer更新对不同类型内容的影响存在差异。，该工具通过Claude token counting API实现，允许用户在实际使用前预估不同模型的token消耗与成本差异，为模型选择提供了实用的量化参考。

Figma的困境因Claude Design而加剧。Figma正日益成为所谓“SaaSpocalypse”受害者案例研究的典型。上周Claude Design的发布，为其增添了全新的痛苦维度。，Figma曾是一款革命性产品。它在2010年代中期推出，利用新兴的WebGL和asm.js技术，首次实现了设计师完全在浏览器中进行设计，开创了全新的SaaS类别。它迅速超越Sketch，成为市场事实上的设计工具，不仅用于UI/UX线框图和原型设计，也越来越多地用于所有平面设计工作。基于浏览器的特性使其具有巨大优势：开发者无需Mac也能打开UI/UX文件（Sketch仅限Mac），同时其协作功能允许用户无需下载任何软件即可实时评论和协作，这使得产品经理和高管等非设计角色也能参与设计迭代，彻底改变了以往通过容易过时的截图文件来回发送修订和注释的低效流程。，然而，LLM的出现使这家极具前瞻性的SaaS公司变得异常脆弱。一个出乎意料的进展是，LLM在设计方面开始变得相当“出色”。虽然目前远不及优秀设计师的水平，但许多辅助性设计工作（如报告、提案等）并不需要顶尖设计人才，而这部分工作往往消耗大量时间。这正是Figma几乎独有脆弱性的根源：它通过吸引非设计师用户而在组织内扩张的模式，一旦这些非设计师用户能够通过AI代理完成设计，就变成了 liability。Figma的S1文件（数据虽已过时，但是唯一可找到的公开细分数据）证实了这一潜在弱点：2025年第一季度，其用户群中只有33%是设计师。，Claude Design的推出直接瞄准了这一软肋。它允许用户通过自然语言提示生成完整、可编辑的设计，其输出质量对于许多内部和辅助设计任务而言已经足够。虽然目前可能还无法替代复杂的产品核心体验设计，但它极大地降低了非专业用户进行设计的门槛。这预示着Figma的增长引擎——向非设计师用户扩张——可能面临严重威胁。Figma的困境不仅来自直接竞争，更源于其核心用户场景可能被AI工具大规模侵蚀。

DeepGEMM是由deepseek-ai开源的一个统一、高性能的张量核心（Tensor Core）内核库。它将现代大语言模型（LLM）的关键计算原语集成到一个简洁、统一的CUDA代码库中，核心目标是提供干净、高效的FP8通用矩阵乘法（GEMM）内核，并支持细粒度缩放。，该库的核心功能与技术特点包括：支持多种精度计算，涵盖FP8、FP4和BF16格式的GEMM运算；集成了融合了重叠通信的混合专家模型（fused MoE with overlapped communication，即Mega MoE）；为DeepSeek v3.2的lightning indexer提供了MQA评分内核（加权ReLU MQA logits）；并包含了HyperConnection（HC）等其他关键组件。所有内核均通过一个轻量级的即时编译（JIT）模块在运行时编译，安装过程中无需CUDA编译，简化了部署流程。，DeepGEMM的设计理念强调简洁与高效。它借鉴了CUTLASS和CuTe的一些概念，但避免重度依赖其模板或代数体系。库中仅包含有限数量的核心内核函数，代码清晰，旨在成为学习和研究NVIDIA GPU内核优化技术的优质资源。尽管设计轻量，但其性能在各种矩阵形状下均能达到或超越专家手动调优的库。，近期的重要更新包括：支持Mega MoE、FP8xFP4 GEMM、FP4索引器以及更快的JIT编译；增加了对SM90/SM100架构的支持，并进行了代码重构以降低JIT CPP模块的CPU开销；提供了稠密和MoE反向传播的权重梯度内核；以及通过启用NVRTC实现了高达10倍的编译加速（可通过设置DG_JIT_USE_NVRTC=1启用，但在某些情况下可能带来性能损失）。据报告，DeepGEMM在H800 GPU上最高可实现1550 TFLOPS的性能表现。

瑞士AI倡议于2023年12月启动，是全球范围内针对AI基础模型的最大规模开放科学/开源项目，也是瑞士国家AI研究所的首个倡议。该研究所由苏黎世联邦理工学院AI中心和洛桑联邦理工学院AI中心合作建立。，该倡议获得了ETH Domain提供的2000万瑞士法郎资助，并获得了由瑞士国家超级计算中心运营的超级计算机“阿尔卑斯”超过1000万GPU小时的初始计算资源支持。“阿尔卑斯”配备了超过1万个GH200 GPU，是世界领先的AI超算之一。倡议汇聚了来自瑞士10多所学术机构的800多名研究人员（包括70位专注于AI的教授），形成了强大的专业知识集群。，其核心使命是加速基础模型开发核心领域以及具有重大社会意义的目标应用领域中开放科学成果的研发。该倡议定期开放算力申请，以汇聚来自不同组织的研究人员，并提供透明开放的软件、模型和数据发布等成果，使包括中小企业和初创公司在内的瑞士各界利益相关者能够可信地使用这些资源。，目前，该倡议已启动第三轮大型项目征集，意向书提交截止日期为2026年3月16日，完整提案截止日期为2026年3月31日，最终决策和项目启动时间定于2026年7月。
