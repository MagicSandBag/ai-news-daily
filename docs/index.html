让Claude审查自身代码：Skyvern如何通过浏览器自动化将AI编程的QA通过率提升至70%，Skyvern是一个用于自动化重复性浏览器操作（如从供应商门户拉取发票、填写医疗表单、从无API的网站提取数据）的工具。团队在使用Claude生成前端代码时，常遇到代码看似正确、类型检查通过，但实际测试时出现按钮无法点击、表单溢出或UI细节不一致等问题。传统的人工QA过程耗时且低效。，为此，团队开发了一个解决方案：利用Skyvern自动化QA流程。他们构建了一个包含33种浏览器工具（会话管理、导航、表单填写、数据提取、凭证管理、工作流等）的MCP服务器，并将其集成到Claude Code中。现在，Claude在每次前端更改后，能自动打开页面、检查像素并运行交互来验证自身工作。，这一改进显著提升了效率：代码更改的一次性通过率从约30%提高到约70%，QA循环时间缩短了一半。团队将此功能封装为两个技能：本地使用的 `/qa` 和持续集成（CI）中使用的 `/smoke-test`。用户可通过 `pip install skyvern` 和 `skyvern setup claude-code` 进行安装体验。该工具会读取git diff、生成测试用例、运行浏览器测试，并输出包含PASS/FAIL结果的表格。整个提示词约700行，已在GitHub开源。，实现过程并非简单地提示“审查你的工作”，而是分为几个阶段：通过git diff理解变更；将变更分类为前端、后端或混合类型；确定验证策略（如决定需要运行多少开发服务器）；对受影响区域运行完整QA（后端和通过本地浏览器的前端测试）；报告结果，并可选择将证据发布到PR。示例输出包括测试项、结果和备注。，在CI流水线中运行的 `/smoke-test` 技能进一步实现了自动化：GitHub Action在PR上触发，分析diff以确定需要测试的范围，在最小化环境中启动应用，并运行基于浏览器的冒烟测试，结果保存至Skyvern供审查。该技能约300行，同样已开源。，这一创新使AI编程助手不仅能生成代码，还能主动验证其正确性，大幅提升了开发流程的可靠性和效率。

Imbue团队发布了一项关于并行测试100多个Claude智能体的案例研究，详细介绍了他们如何利用自研工具`mngr`来创建并改进其自身的测试与文档。这项研究展示了利用大规模并行AI代理进行软件开发和测试验证的新范式。，核心流程始于一个包含命令块的教程脚本`tutorial.sh`。每个命令块由一系列连续的非空行组成。系统首先将每个教程块转换成一个或多个pytest测试函数。随后，为每个pytest函数启动一个独立的Claude智能体，负责执行、调试、修复并改进该测试。最终，整合所有智能体的输出结果。，教程脚本的初始内容由团队编写，但当示例数量超过50个后，人工编写变得繁琐。因此，团队采用了高效的方法：在文件中写入注释（如`# Managing snapshots`），然后请求一个编码智能体根据代码库中已有的文档（包括已提交至Git仓库的自动生成的手册页）来填充具体示例。这一过程产生了双重效益：要么生成高质量示例，要么因生成效果不佳而暴露出`mngr`接口设计或文档的不足，从而驱动团队优化工具，使其界面尽可能简洁直观。，在将教程块转换为pytest函数时，有几个关键细节。教程块通常简洁，有时甚至有些刻意简化，但测试需要更全面，覆盖正常与异常路径。因此，这是一个1:N的对应关系：同一教程块可能因命令变体或环境差异导致不同结果，这些情况往往需要独立的测试用例。为了保持教程块与测试函数之间的可追溯性，系统要求智能体在生成的函数中，通过测试夹具中的特定API引用来“声明”其对应的教程块。此外，还使用简单脚本进行校验，确保对应关系准确无误。，整个架构体现了高度的软件可组合性与可扩展性。通过并行运行大量智能体，`mngr`不仅自动化了测试生成与执行，还将测试过程本身转化为驱动产品改进的反馈循环。这项实践预示了软件开发的未来方向：AI代理不仅能协助编码，还能通过大规模、并行的探索与验证，深度参与系统的质量保障与迭代优化过程。

LLM Wiki 是一个关于如何使用大型语言模型（LLM）构建个人知识库的“想法文件”示例，由知名AI研究员Andrej Karpathy在GitHub上以Gist形式分享。该项目提出了一种超越传统检索增强生成（RAG）模式的知识管理新范式。，其核心理念在于，当前多数基于LLM的文档交互（如NotebookLM、ChatGPT文件上传及常见RAG系统）采用“查询时检索”模式：用户上传文件集，LLM在每次提问时检索相关文本片段并生成答案。这种方式下，LLM每次都需要从零开始重新发现知识，缺乏积累。对于需要综合多份文档信息的复杂问题，模型必须反复查找和拼凑片段，无法构建持续演化的知识体系。，LLM Wiki 提出的方案则截然不同。它旨在让LLM增量式地构建和维护一个持久化的维基（Wiki）——一个位于用户与原始资料之间的、结构化的、相互链接的Markdown文件集合。当用户添加新的资料源时，LLM不仅仅是为后续检索建立索引，而是会主动阅读内容、提取关键信息，并将其整合到现有的维基中。具体操作包括更新实体页面、修订主题摘要、标注新旧数据之间的矛盾之处，从而强化或挑战不断演化的知识综合体。，这种方法的关键价值在于知识被“编译一次，持续更新”，而非每次查询时重新推导。该维基作为一个持久化的、可复合增长的知识中间层，允许知识持续积累和精炼。这个想法文件本身设计用于被复制粘贴到用户自己的LLM智能体（如OpenAI Codex、Claude Code等）中，其目标是传达高层概念，具体的实现细节将由智能体与用户协作完成。
