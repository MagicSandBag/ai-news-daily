本文探讨了如何通过自动化验证机制，使未经人工审查的AI生成代码能够安全地应用于生产环境。作者通过一个简化版FizzBuzz问题的实验，将原本“必须逐行审查AI生成代码”的观念转变为“必须验证AI生成代码的正确性”。这里的“验证”指通过审查、机器可执行的约束条件或两者结合的方式来确认代码正确性，而不仅仅是人工阅读代码。，实验使用编码代理生成解决方案，并迭代检查其是否符合多项预定义约束：1）代码必须通过基于属性的测试，以约束解空间并确保满足需求，包括验证无异常抛出和延迟足够低的测试；2）代码必须通过变异测试，在假设测试正确的前提下，用其限制代码行为，确保仅满足需求；3）代码必须无副作用；4）在Python环境下强制执行类型检查和代码规范检查（其他语言可能不需要）。作者认为这些检查足以让人信任生成的代码而无需查看其内容，尽管仍存在无效但能通过检查的程序空间，但这一空间很小且不易意外进入。，针对可维护性问题，作者提出应将AI生成代码视为编译后的代码，其可读性和可维护性在此场景下可能不相关。目前设置这些约束的开销仍高于直接阅读代码，但随着AI代理和工具改进，这一基线将不断优化。实验代码已在GitHub仓库fizzbuzz-within-human-review中实现，供读者自行尝试。，文中还提及相关研究：形式化验证虽能确保AI生成代码正确，但实践成本高昂；JustHTML项目通过大量单元测试约束AI生成代码，早期版本存在代码“完全镜像测试数据”的问题，但随着LLM模型改进已减少，而基于属性的测试可有效抑制此策略，降低人工审查需求。此外，有团队正在构建无需人工审查代码的“软件工厂”，其具体技术细节尚未深入探讨。整体上，自动化验证为AI生成代码的可靠使用提供了可行路径。

加州大学尔湾分校的研究人员开发出一种名为FlyTrap的新型物理攻击方法，能够利用特制的“对抗性雨伞”诱使基于摄像头的自主目标跟踪（ATT）系统（特别是无人机）危险地接近攻击者，从而使其易于被捕获、遭受传感器攻击甚至发生物理碰撞。这项研究针对广泛应用于监控、边境管控和执法等领域，但也可能被滥用于跟踪与破坏的ATT无人机，揭示了其关键安全漏洞。，该攻击被定义为“距离牵引攻击”（DPA），其核心在于利用ATT系统的视觉跟踪算法缺陷。研究人员设计的FlyTrap攻击框架，使用一把经过特殊图案喷涂的雨伞作为可部署的、针对特定领域的攻击载体。该雨伞上的对抗性图案能够在无人机摄像头的视野中产生误导性信号。，FlyTrap的创新性体现在三个方面：物理可部署性、闭环有效性以及时空一致性。它通过一种新颖的渐进式距离牵引策略和可控的时空一致性设计，在真实世界环境中操纵ATT无人机。攻击过程中，雨伞上的图案会欺骗无人机的跟踪算法，使其误判与目标的距离，从而持续主动地飞向攻击者，将跟踪距离缩短至危险范围。，研究团队进行了全面评估，包括构建新的数据集、设计评估指标，并在真实世界设置中对白盒甚至商用ATT无人机（如大疆DJI和HoverAir的产品）进行了闭环实验。结果表明，FlyTrap能够成功地将无人机的跟踪距离显著减小，使其进入可被物理捕获、传感器易受攻击或直接导致碰撞的范围内。这突显了当前ATT系统面临紧迫的安全风险，对其安全部署具有重要的现实警示意义。该论文的扩展版本已被网络安全顶级会议NDSS 2026接收。
