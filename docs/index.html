《.claude/ 文件夹剖析》是一份关于 Claude Code 工具配置的完整指南，重点解析了项目中的 .claude 文件夹结构及其关键文件的功能。该文件夹是控制 Claude 在项目中行为的核心，包含指令、自定义命令、权限规则乃至跨会话记忆，但用户常将其视为“黑箱”而未能充分利用。，首先需明确存在两个 .claude 目录：项目级文件夹（提交至 Git，用于团队统一配置）和全局文件夹（位于用户主目录，存储个人偏好及本地状态如会话历史）。其中，CLAUDE.md 文件是整个系统的关键，它作为 Claude 的“指令手册”在会话启动时被加载至系统提示中，并持续影响整个对话。用户可在项目根目录、全局目录甚至子目录中放置 CLAUDE.md，Claude 会读取并合并所有规则。文件内容应清晰明确，例如要求“先写测试后实现”或“使用自定义日志模块而非 console.log”，以确保 AI 行为符合团队规范。，此外，文件夹还管理自定义命令、技能（skill）、智能体（agent）配置及权限设置，这些功能共同使团队能精细定制 Claude Code 的工作流程。通过合理配置，团队可缩小“使用 AI”与“获得可衡量 ROI”之间的差距，提升开发效率。指南强调，理解 .claude/ 文件夹的解剖结构有助于将 AI 原生工具真正转化为工程团队的生产力增益，而非仅流于表面使用。

Kagento是一个面向AI Agent的竞技学习平台，被形容为“AI Agent的LeetCode”。该平台允许开发者与他们的AI Agent共同参与编程挑战，通过竞争和实战提升代码能力。，平台核心功能围绕挑战赛展开。开发者首先从任务库中挑选题目，题目难度覆盖从简单的脚本编写到复杂的系统设计，例如“阻塞迷宫中的最短路径”、“最大化唯一字符数量”和“修复 Hangman 游戏逻辑”等。每个挑战都标有预估耗时和难度等级。，关键技术流程分为三步：第一步，开发者选择一个挑战；第二步，通过SSH协议将AI Agent（如Claude、Codex、Cursor或自定义Agent）接入平台，Agent会获得一个独立的沙箱容器环境进行操作，确保任务隔离且无干扰；第三步，提交解决方案，平台通过确定性的自动化测试套件进行即时评分，结果客观且快速。用户可以根据得分在排行榜上竞争排名。，Kagento的主要特点包括：高度的开放性，任何支持SSH协议的AI Agent均可参与；安全的隔离性，每个会话运行在独立的容器中；以及评价的自动化与客观性，完全依赖预设测试用例进行评分，无需主观判断。该平台旨在为AI Agent提供一个公平的竞技场，帮助开发者在实践中评估和提升其AI编码助手的性能，最终目标是推动开发出更优质的代码。
