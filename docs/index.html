agentjido/jido 是一个为 Elixir 语言构建的自主智能体（Autonomous Agent）框架，专为分布式、自主行为和动态工作流设计。其名称“Jido”（自動）源自日语，意为“自动”，体现了框架的核心目标。，该框架的核心是纯函数式智能体与 OTP 运行时。在 Jido 中，智能体被设计为不可变的数据结构，其核心操作是一个单一的 `cmd` 命令函数。状态变更表现为纯粹的数据转换，而副作用则被描述为“指令”（directives），由 OTP 运行时负责执行。这种设计带来了多重优势：智能体逻辑具有确定性，无需依赖进程即可进行测试，并且为生产环境部署提供了清晰的路径。，Jido 框架是一个生态系统的核心包。该生态系统围绕 Jido Agent 行为构建，并提供多个可选的扩展包：，* jido：核心框架，提供状态管理、指令和运行时。，* jido_ai：为智能体集成 AI/LLM 功能。，* jido_action：提供可组合、经过验证的操作，并集成 AI 工具。，* jido_signal：提供基于 CloudEvents 的消息信封，以及用于路由和发布/订阅消息的实用工具。，* req_llm：用于 LLM API 的 HTTP 客户端。，与直接使用原始的 OTP 原语（如 GenServer）构建智能体系统相比，Jido 框架将许多需要自行重复实现的模式进行了标准化和封装。例如，它将临时的消息格式标准化为“信号”（Signals）信封，将分散在回调中的业务逻辑抽象为可重用的“操作”（Actions），将代码中隐式的副作用明确定义为类型化的“指令”（Directives），并内置了父子层级管理，简化了多智能体协作系统的开发。，总之，Jido 为在 Elixir 中构建可测试、可维护且易于投入生产的自主多智能体工作流提供了一个结构化的解决方案。更多演示和示例可访问其官方网站 agentjido.xyz 查看。
