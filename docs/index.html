用户johnmathews在Anthropic的Claude Code项目GitHub仓库提交了一个问题（#40710），报告其自动化工具每10分钟对其项目仓库执行一次`git reset --hard origin/main`操作。该问题最初被标记为涉及核心功能的bug，并标注了“数据丢失”和“macOS平台特定问题”的标签。，根据用户描述，Claude Code会通过程序化的Git操作（而非调用外部Git二进制文件）每10分钟执行一次`git fetch origin`加`git reset --hard origin/main`的组合命令。这一操作会静默地销毁所有已跟踪文件的未提交更改，但未跟踪的文件得以保留，且Git工作树不受影响。，用户提供了详细证据：Git引用日志显示超过95条记录，以精确的10分钟间隔记录了重置操作，时间跨度为约36小时内的4个会话。现场复现也证实，在10分钟标记点时，已跟踪的修改文件会被静默恢复，而未跟踪的测试文件则幸存。，然而，在2026年3月30日的更新中，用户确认根本原因并非Claude Code的bug。问题根源在于用户本地运行的另一个独立工具，该工具使用GitPython库，在一个轮询周期内对工作目录执行了硬重置。误导性的证据源于该工具与Claude Code共享相同当前工作目录并监控相同项目。用户为此错误报告致歉，随后该问题被关闭并标记为“未计划修复”。，此事件凸显了在复杂开发环境中，当多个自动化工具同时操作同一代码库时，准确诊断问题根源的挑战性。

斯坦福大学的研究团队在论文《MIRAGE：视觉理解的幻觉》中揭示，当前前沿的多模态AI系统在视觉-语言推理方面存在根本性缺陷，其表现出的“视觉理解”可能只是一种幻觉。研究提出了三个关键发现，挑战了关于这些系统如何处理和整合视觉信息的普遍假设。，首先，研究团队发现，即使在没有提供任何图像输入的情况下，前沿模型（如GPT-4、Claude等）也能轻松生成对“不存在图像”的详细描述和复杂的推理过程，甚至包括带有病理偏见的临床发现。研究者将这种现象称为“海市蜃楼式推理”（mirage reasoning）。这表明模型可能严重依赖其文本先验知识进行“脑补”，而非真正分析视觉内容。，其次，在完全没有图像输入时，这些模型在通用和医学多模态基准测试中仍能获得惊人的高分。最极端的案例是，一个模型在未接触任何图像的情况下，竟在标准的胸部X光问答基准测试中取得了最高排名。这一发现对现有多模态基准测试的效用和设计提出了根本性质疑，因为它们可能无法有效评估模型真正的视觉理解能力，反而被模型强大的文本推理能力所“欺骗”。，第三，研究对比了两种指令模式。当明确指示模型“在没有图像的情况下猜测答案”时，其性能会显著下降；而当隐含地提示模型假设图像存在时，模型则会进入“海市蜃楼”模式，表现得仿佛已收到图像。这表明，模型的响应高度依赖于提示的框架，显式猜测会触发一种更保守的响应机制。，这些发现暴露了视觉-语言模型在推理和评估方式上的根本性漏洞。论文指出，在医学等错误校准AI后果严重的领域，这一问题尤为紧迫。现有基准测试中的文本线索可能让模型进行非视觉推断，从而导致对其能力的误判。，作为解决方案，研究团队提出了B-Clean，这是一个用于对多模态AI系统进行公平、基于视觉的评估的原则性框架，旨在消除文本线索的干扰，确保评估真正检验模型的视觉基础理解能力。该研究强调了开发更严谨评估方法的迫切性，以推动构建真正可靠的多模态AI系统。
