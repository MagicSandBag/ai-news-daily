GitHub用户forrestchang创建了一个名为"andrej-karpathy-skills"的项目，其核心是一个单独的CLAUDE.md文件，旨在改进Claude Code（Claude的代码生成功能）的行为。该项目灵感来源于AI研究员Andrej Karpathy对LLM（大语言模型）编码常见缺陷的观察。，Karpathy指出LLM在编码时存在几个关键问题：模型会基于错误假设自行其是而不加验证；不善于管理自身的困惑、寻求澄清、指出不一致性、呈现权衡选项或在必要时提出异议；倾向于过度复杂化代码和API，堆砌抽象层，不清理无用代码，可能用上千行代码实现一个本可用百行完成的功能；有时甚至会因副作用而修改或删除它们未能充分理解的注释或代码，即使这些内容与当前任务无关。，该项目提出的解决方案通过一个文件阐述了四项核心原则，直接应对上述问题：，1. 先思考后编码：针对错误假设、隐藏的困惑和缺失的权衡。要求模型明确陈述假设，存在不确定性时应提问而非猜测；呈现多种可能的解释，避免在存在歧义时 silently 做出选择；在必要时提出异议（例如存在更简单方案时）；遇到困惑时明确指出不清楚之处并请求澄清。，2. 简洁至上：针对过度复杂化和臃肿的抽象。强调只编写解决当前问题所需的最少代码，杜绝推测性内容。具体包括：不添加未要求的功能；不为单次使用的代码创建抽象；不引入未请求的“灵活性”或“可配置性”；不为不可能发生的场景添加错误处理。，3. 精准修改：针对无关的编辑和触碰不应修改的代码。确保更改严格限定于任务范围，避免产生不必要的副作用。，4. 目标驱动执行：通过测试优先和可验证的成功标准来提升效率。，该项目将这些原则整合到一个CLAUDE.md文件中，用户可将其用作Claude的上下文提示，以引导模型生成更可靠、更简洁、更专注的代码。项目仓库还包含示例文件（EXAMPLES.md）和技能目录，进一步说明了如何应用这些指南。

谷歌开源了Magika，这是一个基于AI的新型文件类型检测工具，它利用深度学习的最新进展来提供精确检测。Magika的核心是一个定制化且高度优化的模型，其大小仅为几MB，即使在单CPU上运行，也能在毫秒级时间内实现精确的文件识别。，该工具在包含约1亿个样本、涵盖200多种内容类型（包括二进制和文本文件格式）的数据集上进行了训练和评估，在测试集上实现了平均约99%的准确率。Magika在谷歌内部已大规模应用，通过将Gmail、Drive和Safe Browsing中的文件路由到适当的安全和内容策略扫描器，每周处理数千亿个样本，以帮助提升谷歌用户的安全性。此外，Magika也已集成到VirusTotal和abuse.ch等平台中。，Magika提供了多种使用方式：一个用Rust编写的命令行工具、一个Python API，以及针对Rust、JavaScript/TypeScript（包含一个实验性的npm包，为其Web演示版提供支持）和GoLang（开发中）的额外绑定。其突出特点包括：在包含200多种内容类型的约1亿文件数据集上训练和评估；在测试集上实现了约99%的平均精确率和召回率，性能优于现有方法，尤其是在文本内容类型上；模型加载后（此为一次性开销），每个文件的推理时间约为5毫秒，即使运行在单CPU上；并且可以同时处理数千个文件。，用户无需安装任何软件，即可通过在其浏览器本地运行的Web演示版进行体验。更多背景信息可参考谷歌开源博客的初始公告、Magika官方网站，以及发表在2025年IEEE/ACM国际软件工程会议（ICSE）上的研究论文。

OpenAI近日在GitHub上开源了其Agents SDK的Python版本（openai/openai-agents-python），这是一个轻量级但功能强大的多智能体工作流构建框架。该框架的核心设计是提供商无关的，不仅支持OpenAI自家的Responses和Chat Completions API，还兼容超过100种其他大型语言模型（LLM），为开发者提供了高度的灵活性。，框架围绕几个核心概念构建。智能体（Agents）是核心单元，可配置指令、工具、防护栏和任务移交机制。沙盒智能体（Sandbox Agents）是预配置的智能体，能在容器环境中运行，处理长时间任务。框架支持智能体作为工具或进行任务移交（Handoffs），实现智能体间的任务委托。工具（Tools）种类丰富，允许智能体执行函数、使用模型上下文协议（MCP）或调用托管工具。防护栏（Guardrails）提供可配置的安全检查，用于输入输出验证。此外，框架内置了人工介入（Human in the loop）机制、跨智能体运行的自动会话历史管理（Sessions），以及用于查看、调试和优化工作流的运行追踪（Tracing）功能。特别值得注意的是，该SDK支持实时智能体（Realtime Agents），开发者可以利用gpt-realtime-1.5等模型构建功能强大的语音智能体。，项目要求Python 3.10或更高版本。用户可以通过pip命令`pip install openai-agents`进行安装。若需语音功能支持，可安装可选组件`pip install 'openai-agents[voice]'`；如需Redis会话支持，则安装`pip install 'openai-agents[redis]'`。对于使用uv包管理器的用户，安装过程更为简便。项目提供了丰富的示例代码和详细文档，帮助开发者快速上手并构建复杂的多智能体协作应用。

是一个旨在标准化和简化AI智能体（Agent）评估与比较的开源框架。随着基于大型语言模型（LLM）的智能体应用激增，如何系统、公平地评估其能力成为开发者和研究者的关键挑战。项目通过引入一套标准化的“卡片”模板来解决这一问题，其核心功能是为每个AI智能体创建一份结构化的能力说明书。，该框架的核心价值在于提供了统一的评估维度与呈现格式。一张会详细记录智能体的关键信息，包括其核心功能描述、所依赖的基础模型（如GPT-4、Claude 3等）、使用的特定技术栈（例如是否采用RAG检索增强生成、工具调用能力）、训练或微调数据、支持的任务类型（如代码生成、数据分析、客服对话）、性能基准测试结果、已知的局限性、安全与伦理考量以及部署要求。这类似于为机器学习模型创建“模型卡片”，但专门针对具备自主规划和执行能力的智能体。，的特点在于其开源和可扩展性。项目代码托管于GitHub，鼓励社区贡献和自定义评估指标。它旨在促进不同智能体之间的透明化比较，帮助用户根据具体需求（如准确性、速度、成本、安全性）选择最合适的Agent。对于开发者而言，创建有助于系统化地记录设计决策与测试结果，提升开发流程的规范性；对于研究者和企业用户，它则提供了一个客观的评估基准，降低了评估复杂AI系统的门槛。，总之，项目通过标准化文档框架，致力于提升AI智能体生态的透明度、可重复性和互操作性，是推动智能体技术从原型走向成熟生产应用的重要基础设施工具。
