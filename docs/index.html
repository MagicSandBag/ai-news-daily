GitHub用户forrestchang创建了一个名为"andrej-karpathy-skills"的项目，其核心是一个单独的CLAUDE.md文件，旨在优化Claude Code（Anthropic的AI编程助手）的行为。该项目灵感来源于AI研究员Andrej Karpathy对大型语言模型（LLM）在编程任务中常见缺陷的观察。，Karpathy指出LLM在编码时存在几个关键问题：模型会基于错误假设自行推进而不加验证；不善于管理困惑、寻求澄清、揭示矛盾、呈现权衡或在必要时提出异议；倾向于过度复杂化代码和API，引入臃肿的抽象层，不清理无用代码；有时甚至会侧向地修改或删除它们未能充分理解的注释或代码。，该项目提出的解决方案浓缩为四项核心原则，并集成于CLAUDE.md文件中，以直接应对上述问题：，1. 先思考后编码：针对错误假设和隐藏的困惑。要求模型明确陈述假设，在不确定时主动提问而非猜测；呈现多种可能的解释，不 silently 做出选择；当存在更简单方案时提出异议；遇到困惑时明确指出不清楚之处并请求澄清。，2. 简洁至上：针对过度复杂化和抽象膨胀。坚持用最少的代码解决问题，不添加未要求的功能、单次使用的抽象、未请求的“灵活性”或“可配置性”，以及为不可能场景进行的错误处理。如果代码能从200行精简至50行，就应重写。评判标准是：资深工程师是否会认为这过于复杂？若是，则简化。，3. 精准修改：针对无关的编辑和不应触碰的代码。严格限定只修改必须更改的部分，避免产生不必要的副作用。，4. 目标驱动执行：通过测试先行和可验证的成功标准来提升效率。，该项目通过这个集中的指导文件，为Claude Code提供了结构化的行为准则，旨在引导其生成更可靠、更简洁、更专注的代码，减少LLM典型的臆断和过度工程化倾向，从而提升AI编程助手的实用性和输出质量。

Ralph 是一个自主运行的 AI 代理循环，其核心设计是重复运行 AI 编码工具（默认为 Amp CLI 或可选的 Claude Code），直至产品需求文档（PRD）中的所有项目均被完成。该工具基于 Geoffrey Huntley 提出的 Ralph 模式，其关键特点是每次迭代都是一个具有全新、干净上下文的新实例，而项目记忆与进度则通过 git 历史、`progress.txt` 文件和 `prd.json` 文件来持久化保存。，项目提供了灵活的安装与配置方式。用户可以选择将 `ralph.sh` 脚本及相关提示模板复制到自己的项目目录中，也可以将配套的“技能”全局安装到 Amp 或 Claude Code 的配置目录下，以便在所有项目中使用。对于 Claude Code 用户，还可以通过插件市场直接添加 Ralph 市场并安装技能。安装后，可用的技能包括 `/prd`（用于生成产品需求文档）和 `/ralph`（用于将 PRD 转换为 `prd.json` 格式）。当用户向 Claude 发出“创建 prd”、“为此功能编写 prd”或“转换为 ralph 格式”等指令时，这些技能会被自动调用。，为了优化使用体验，项目推荐为 Amp 工具配置自动切换（auto-handoff）功能。这通过在用户设置文件中添加相应配置项来实现，其作用是当上下文即将填满时自动进行切换，从而确保长周期、多迭代的自主编码任务能够顺畅运行，无需人工干预。Ralph 通过将高层级的 PRD 逐步转化为具体的代码实现，旨在自动化软件开发周期中的部分环节。
