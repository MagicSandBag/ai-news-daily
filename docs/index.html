GitHub用户lingfengQAQ开源了名为“webnovel-writer”的长篇网文辅助创作系统。该项目基于Claude Code构建，核心目标是解决当前AI写作中普遍存在的两大难题：长文本“遗忘”（即模型无法记住前文大量细节）和“幻觉”（生成与既定事实或设定矛盾的内容）。系统宣称能够支持高达200万字量级的超长周期连载创作。，该系统通过一系列模块化命令和智能体（Agent）协同工作。用户安装后，可通过`/webnovel-init`初始化一个新的小说项目。核心创作流程由三个主要命令驱动：`/webnovel-plan`用于制定大纲，`/webnovel-write`执行具体章节的写作，`/webnovel-review`则负责对已写内容进行审阅和修订。系统内置了多种Agent，如上下文管理Agent，它们默认继承当前Claude会话的模型，但用户也可通过编辑配置文件单独指定为sonnet、opus或haiku等模型。，为应对“遗忘”问题，项目深度集成了RAG（检索增强生成）技术。用户必须配置RAG环境，包括设置嵌入（Embedding）模型（如Qwen/Qwen3-Embedding-8B）和重排序（Reranker）模型（如jina-reranker-v3）的API端点与密钥。系统利用这些技术从已创作的海量文本中智能检索相关上下文，提供给生成模型，从而保持故事的一致性和连贯性。在v5.4.3版本中，还增强了智能RAG上下文辅助，采用了`auto/graph_hybrid`混合检索策略，并在必要时回退到BM25算法。，项目另一个重要特点是引入了“追读力系统”（自v5.3版本），这是一个模拟读者阅读体验和兴趣度的内部评估机制，包含Hook（钩子）、Cool-point（爽点）、微兑现和债务追踪等概念，旨在辅助作者优化行文节奏和情节吸引力。，此外，系统提供了可选的可视化Dashboard（v5.5.0新增），用户通过`/webnovel-dashboard`命令可启动一个只读面板，用于查看项目状态、实体关系图谱、浏览章节大纲以及追读力数据。该项目采用GPL v3开源协议，并通过官方Claude Plugin Marketplace简化了安装流程。

Jido 2.0 现已发布，这是一个基于 Elixir 语言构建的 AI Agent 框架。经过 18 个月的开发与重构，该版本已正式在 Hex 包管理器上架。，项目起源于 2024 年的 BotHive 机器人平台。随着 AI 浪潮兴起，开发者基于对 BEAM 虚拟机（Erlang 运行时）的坚定信念进行了重构，认为其面向并发的特性和容错能力是构建持久、稳定 Agent 系统的理想基础，优于单线程的 TypeScript 框架或难以长期运行的 Python 方案。，相较于过度设计、使用复杂的 1.0 版本，Jido 2.0 的核心改进在于简化。它提供了更简洁的 API，减少了冗余代码，并彻底贯彻“BEAM 优先”的设计理念。其目标是让开发者专注于构建 Agent 逻辑，而非与框架本身对抗。，框架的核心是一个强健、持久的纯函数式 Agent 架构。Agent 被定义为包含状态、动作和工具的数据结构（struct），可在 GenServer 中运行。所有交互都通过单一的 `cmd/2` 函数进行：输入动作，输出更新后的 Agent 和一系列指令（directives）。Agent 始终保持为纯数据，副作用则被描述为由运行时执行的、类型化的指令。这种设计使得 Agent 的逻辑更易于推理、测试和调试，开发者可以无需连接网络、数据库或 LLM 即可单元测试 Agent 的每个决策。，Jido.AgentServer 将任何 Agent 包装在一个受监督的 GenServer 中，提供了信号路由、指令执行和父子 Agent 层级管理功能，构成了完整的 Agent 运行时。，策略（Strategies）是关键扩展点，用于控制 `cmd/2` 如何处理动作，且支持插件化。核心框架内置两种策略：用于顺序执行的 Direct 策略，以及带有状态转换守卫的有限状态机（FSM）策略。这些策略无需 AI 参与即可覆盖大量应用场景。Jido AI 功能也通过此方式接入，支持 ReAct、思维链等模式。，总之，Jido 2.0 是一个充分利用 Elixir 和 BEAM 虚拟机并发与容错优势的 Agent 框架，其纯函数式、数据驱动的核心设计旨在提升开发体验与系统可靠性。
