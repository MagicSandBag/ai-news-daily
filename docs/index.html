Cook是一个用于编排Claude Code、Codex和OpenCode的简单命令行界面（CLI）工具，旨在通过结构化的工作流循环提升AI编程任务的效率。它基于三类基本令牌构建工作流：核心工作单元（Work）、循环操作符（Loop operators）和组合操作符（Composition operators）。，核心功能围绕循环操作符展开，包括重复执行（xN）、审查循环（review）和任务列表推进（ralph）。xN操作符可将工作单元顺序执行N次，每次迭代都能看到前一次的输出。review操作符引入了一个质量审查循环：工作完成后，审查者评估输出，网关决定完成（DONE）或迭代（ITERATE）；若需迭代，系统执行修复步骤后重复审查流程。用户可自定义审查、网关和迭代提示词，并为各步骤指定不同的AI代理或模型（如Codex、GPT-5-Codex、Claude Opus）。ralph操作符则在外层添加网关，用于管理任务列表的推进，使工作提示能根据项目状态自主定位当前任务。，组合操作符支持并行执行多个Cook实例并在隔离的git工作树中运行，最后通过解析器整合结果。versions（vN/race N）操作符并行运行N个相同的工作流，默认使用pick解析器选择最佳输出，也可指定选择标准（如“代码量最少”）。vs操作符允许并行运行两个不同的工作流分支。，操作符遵循从左到右的组合逻辑，每个操作符会包装其左侧的所有内容，从而灵活构建复杂工作流。例如，cook "Add dark mode" x3 review表示先顺序执行三次工作单元，再进行一次审查循环；而cook "Add dark mode" review v3 pick则表示并行运行三个带审查循环的实例，并挑选最佳结果。，该工具通过模块化设计简化了AI编程任务的编排，支持自定义提示、模型选择和迭代控制，适用于代码生成、审查与优化等场景，提升了开发流程的自动化程度与可靠性。
