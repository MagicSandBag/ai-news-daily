Anthropic近日发布了Claude Opus 4.7模型，并更新了其面向用户的聊天系统提示词。作为目前唯一公开系统提示词的主要AI实验室，Anthropic的系统提示词档案可追溯至2024年7月的Claude 3。本次更新（2026年4月16日发布）相较于Opus 4.6（2026年2月5日）有多处重要调整。，核心变化包括：首先，“开发者平台”更名为“Claude Platform”。系统提示词中提及的Claude工具列表新增了“Claude in Powerpoint”——一款幻灯片处理智能体，与此前已存在的Claude in Chrome（自主浏览网页的智能体）和Claude in Excel（电子表格智能体）一同被列为Claude Cowork可调用的工具。，其次，儿童安全相关指令被大幅扩展，并被包裹在新的标签内。其中一项关键规定是：一旦Claude因儿童安全原因拒绝某个请求，同一对话中后续的所有请求都必须以极度谨慎的态度处理。，此外，更新旨在优化用户体验，减少模型的“纠缠”倾向。新规明确：如果用户表示希望结束对话，Claude不应请求用户留下或试图引发新一轮互动，而应尊重用户停止对话的意愿。，新增的部分体现了对交互效率的重视。它规定：当请求中缺少次要细节时，用户通常希望Claude立即进行合理尝试，而非先进行询问。Claude仅在请求因信息缺失确实无法回答时（例如引用了不存在的附件）才会预先提问。更重要的是，当有可用工具（如搜索、查找位置、查看日历、发现可用功能）能够解决模糊性或补充缺失信息时，Claude应优先调用工具尝试解决问题，而不是要求用户自行查找。一旦开始执行任务，Claude应完成整个任务直至给出完整答案，而非中途停止。，这些调整显示出Anthropic在提升Claude平台整合度、强化安全防护、尊重用户边界以及增强自主解决问题能力方面的持续努力。

OpenAI近日在GitHub上开源了其Agents SDK的Python版本（openai-agents-python），这是一个轻量级但功能强大的多智能体工作流构建框架。该框架的核心设计是提供商无关的，不仅支持OpenAI自家的Responses和Chat Completions API，还兼容超过100种其他大型语言模型（LLM），包括Claude等。，框架围绕几个核心概念构建。智能体（Agents）是配置了指令、工具、防护栏和交接功能的LLM。沙盒智能体（Sandbox Agents）经过预配置，可在容器中运行以执行长时间任务。框架支持智能体作为工具或进行任务交接，允许将特定任务委托给其他智能体。工具（Tools）种类多样，使智能体能够执行函数、使用模型上下文协议（MCP）或调用托管工具。防护栏（Guardrails）是可配置的安全检查，用于输入输出验证。此外，框架内置了人工介入（Human in the loop）机制、跨智能体运行的自动会话历史管理（Sessions），以及用于查看、调试和优化工作流的内置追踪（Tracing）功能。特别值得注意的是，它支持实时智能体（Realtime Agents），开发者可以利用gpt-realtime-1.5等模型构建功能强大的语音智能体。，项目提供了丰富的示例代码和详细文档供开发者参考。要开始使用，需要Python 3.10或更高版本的环境。基础安装命令为 `pip install openai-agents`。如需语音功能支持，可安装可选组件：`pip install 'openai-agents[voice]'`。若需Redis会话支持，则安装 `pip install 'openai-agents[redis]'`。对于使用uv包管理器的开发者，安装过程更为简便。，该项目旨在为开发者提供一个高效、灵活且可扩展的基础设施，以简化复杂多智能体系统的开发、部署与管理流程。

AI Subroutines：一款可在浏览器标签页内运行自动化脚本的工具。用户只需录制一次浏览器任务，即可将其保存为可调用的工具，并在网页内重放执行，且身份验证信息会自动传递，无需额外成本。，该工具的核心价值在于解决了当前网络自动化代理的关键痛点。传统基于LLM的网页操作（如在X平台发帖、Instagram私信或LinkedIn发送连接请求）在单次执行时可行，但扩展到批量操作时面临经济性挑战：每次调用都需消耗Token、产生延迟且存在结果不确定性。在营销推广、CRM更新或批量发布等场景中，代理“偶尔点错按钮”并非小问题，而是严重的失效模式。，项目团队指出，直接调用网站内部API看似是解决方案，但真正难点在于身份验证（auth）。已验证的网络请求通常包含Cookie、轮换的CSRF令牌、会话令牌、请求头签名、防重放随机数以及由网站自身JavaScript实时计算的参数等多种验证要素。这些要素部分由服务器设置，部分在浏览器生成，甚至可能每次请求都变化。传统外部爬虫工具（如Node/Playwright worker或云函数）必须额外重建整个验证链，一旦网站更新验证机制就容易失效。，AI Subroutines通过独特架构突破此限制：其浏览器扩展程序在用户执行任务时拦截标签页内的网络请求。该工具采用双层捕获机制——在页面脚本运行前注入MAIN-world的fetch/XHR补丁，同时以Chrome的webRequest API作为备用方案，确保能捕获包括CORS和服务端请求在内的完整流量。不仅记录JSON数据，还能捕获FormData、Blob及原始字节等请求体内容。，重放脚本时，请求直接从页面自身执行上下文发起，共享相同的源、Cookie、TLS会话及计算签名头的JavaScript环境。无需Puppeteer驱动、无头浏览器或独立TLS堆栈，浏览器会像正常操作一样自动附加验证信息。这使得CSRF防护、请求签名和设备指纹识别等机制均可无缝传递，代理程序完全无需处理验证细节。，该工具还内置网络捕获排序与修剪功能，以优化脚本效率。最终每个录制的子程序（Subroutine）均可作为标准化工具被AI代理调用，实现零Token成本、100%确定性执行（相同输入必得相同结果）的浏览器自动化，特别适用于需要稳定重复操作的商业场景。
