GitHub用户forrestchang创建了一个名为"andrej-karpathy-skills"的项目，其核心是一个单独的CLAUDE.md文件，旨在优化Claude Code（Anthropic的AI编程助手）的行为。该项目灵感来源于AI研究员Andrej Karpathy对大型语言模型（LLM）在编程任务中常见缺陷的观察。，Karpathy指出LLM在编码时存在几个关键问题：模型会基于错误假设自行推进而不验证；不善于管理困惑、寻求澄清、揭示不一致性、呈现方案权衡或在必要时提出异议；倾向于过度复杂化代码和API，引入臃肿的抽象层，不清理无效代码，可能用上千行代码实现一个本可用百行完成的功能；有时甚至会作为副作用修改或删除它们未能充分理解的注释或代码。，该项目提出的解决方案将这些洞见浓缩为四项核心原则，并集成在一个CLAUDE.md文件中，以直接指导Claude Code的行为：，1. 先思考后编码：针对错误假设和隐藏困惑。要求模型明确陈述假设，在不确定时主动提问而非猜测；当存在歧义时，呈现多种解释而非 silently 选择；在存在更简单方案时提出异议；当感到困惑时，明确指出不清楚之处并请求澄清。，2. 简洁至上：针对过度复杂化和抽象膨胀。倡导用最少的代码解决问题，杜绝 speculative 功能。具体要求包括：不添加未要求的功能；不为单次使用的代码创建抽象；不引入未请求的“灵活性”或“可配置性”；不为不可能发生的场景添加错误处理。，3. 精准修改：针对无关的编辑和误触不应更改的代码。强调修改应严格限定于任务范围，避免产生副作用。在修改代码前，需先理解其上下文和目的。，4. 目标驱动执行：旨在通过测试优先和可验证的成功标准来提升效率。鼓励在编写实现代码前先定义测试或验证方法，确保工作聚焦于达成明确、可检验的目标。，该项目将这些原则具体化为可操作的提示指令，供用户在Claude Code中调用。项目仓库还包含示例文件（EXAMPLES.md）和技能目录（skills/），展示了如何应用这些指南。开发者forrestchang同时推荐了开源平台Multica，用于运行和管理具备可复用技能的编程智能体。该项目的价值在于将专家对LLM编码弱点的分析转化为一套实用、可部署的指导方针，旨在使AI编程助手的行为更加审慎、清晰和高效。
