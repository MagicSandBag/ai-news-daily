近日，安全研究人员披露了FreeBSD内核中的一个高危漏洞CVE-2026-4747。该漏洞存在于`kgssapi.ko`内核模块的RPCSEC_GSS实现中，是一个栈缓冲区溢出漏洞，可导致远程内核代码执行并最终获得root权限的交互式shell。受影响的版本包括FreeBSD 13.5（p11之前）、14.3（p10之前）、14.4（p1之前）和15.0（p5之前）。攻击面为加载了`kgssapi.ko`模块的NFS服务器（端口2049/TCP）。，漏洞的根本原因位于`sys/rpc/rpcsec_gss/svc_rpcsec_gss.c`文件的`svc_rpc_gss_validate()`函数中。该函数在验证GSS-API签名时，会先将32字节的固定RPC头部字段写入一个128字节的栈缓冲区（`rpchdr[]`），然后紧接着将整个RPCSEC_GSS凭证体（`oa_length`字节）复制到剩余空间中。问题在于，复制前缺少对`oa_length`的边界检查。栈缓冲区在存放32字节头部后，仅剩96字节空间用于存放凭证体。如果攻击者构造的凭证体长度超过96字节，就会导致栈缓冲区溢出，覆盖栈上的局部变量、被调用者保存的寄存器以及函数返回地址，从而在后续调用`gss_verify_mic()`之前就实现了控制流劫持。，在修复版本（如14.4-RELEASE-p1）中，补丁在复制操作前添加了边界检查：`if (oa->oa_length > sizeof(rpchdr) - 32) return (FALSE);`，确保凭证体长度不会超出可用空间。，此漏洞的特别之处在于，整个漏洞的发现、分析和利用代码的撰写过程均由Anthropic的AI助手Claude完成。研究人员仅提供了初始的漏洞公告和补丁链接，Claude便自主完成了对补丁的逆向分析、漏洞原理的深入理解，并最终编写出完整的远程内核利用代码。该利用代码能够绕过ASLR（地址空间布局随机化）和SMAP（管理模式访问保护），在目标系统上实现稳定的远程代码执行，并返回一个root权限的交互式shell。这一案例展示了高级LLM在漏洞研究和利用开发领域的潜在能力。
