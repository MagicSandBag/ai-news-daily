谷歌开源了其实验性的多智能体编排测试平台Scion。该平台旨在管理在本地和远程计算环境中容器内并发运行的智能体，使开发者能够运行具有独立身份、凭证和共享工作空间的专用智能体群组。，谷歌将Scion描述为“智能体的虚拟机管理程序”，能够将智能体记忆、聊天室和任务管理等多智能体系统组件作为正交关注点进行集成。Scion将“深度智能体”（如Claude Code、Gemini CLI、Codex等）作为隔离的并发进程进行编排。每个智能体拥有自己的容器、git工作树和凭证，从而可以在项目的不同部分独立工作，互不干扰。智能体可以运行在本地、远程虚拟机或跨Kubernetes集群。，该平台使开发者能够管理一个动态演化、并行执行的任务图，这些任务追求不同的目标，例如编码、审计和测试。它不依赖于固定的智能体集合，而是支持不同的智能体生命周期：有些智能体是专用且长期存活的，而另一些则是临时的，仅与单一任务绑定。，Scion的一个基本原则是优先考虑隔离而非约束，以确保智能体操作安全。这意味着，与其通过定义规则并将其嵌入上下文来约束智能体的行为，Scion选择让智能体自由执行完成任务所需的任何操作，同时在外部设置边界和防护栏：Scion倾向于让智能体在“yolo模式”下运行，同时将它们隔离在容器、git工作树以及受基础设施层网络策略约束的计算节点中。，Scion通过称为“harness”的适配器支持多个流行智能体，这些适配器负责管理生命周期、身份验证和配置。目前已支持的智能体包括Gemini、Claude Code、OpenCode和Codex。该项目采用Apache 2.0许可证开源，代码托管在GitHub上。谷歌强调Scion目前仍处于实验阶段，旨在探索多智能体系统编排模式，而非作为生产就绪解决方案。
