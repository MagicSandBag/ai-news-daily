VS Code Agent Kanban：为AI辅助开发者设计的任务管理工具，英国软件工程师Gareth Brown近日发布了VS Code扩展“Agent Kanban”，旨在解决AI编程助手（如GitHub Copilot）工作流中的核心痛点——上下文丢失（context rot）。该工具已在VS Code Marketplace和GitHub上架。，核心问题是，当前AI编码助手缺乏持久记忆，每次对话都从零开始。这导致长期任务中上下文窗口过度膨胀（context bloat），拖慢LLM响应并增加遗忘早期约束的风险；任务历史、决策依据和计划在清空聊天或关闭VS Code后便永久消失；任务管理通常依赖开发者记忆或外部待办应用，与代码及AI工作流脱节；团队也缺乏对AI工作进度和决策的共享视图。，Agent Kanban通过将任务记录与看板管理深度集成到IDE中来应对这些挑战。其四大核心功能包括：，1. 在VS Code内部集成GitOps与团队友好的看板面板。，2. 通过@kanban命令实现结构化的“计划/待办/执行”流程。，3. 利用开发者现有的AI代理框架，而非捆绑内置方案。，4. 采用.md任务文件格式作为持久化、可编辑的“唯一事实来源”，记录思考、决策和行动，有效抵御上下文腐化。，该工具采用极简设计：每个任务都是一个Markdown文件，存储在.agentkanban/tasks/目录下。文件包含YAML前言，用于记录任务标题、看板状态和時間戳。文件主体则是结构化的对话日志，使用[user]和[agent]标记来记录开发者与AI助手的完整交互过程。这种设计确保了任务上下文、技术决策和实现细节被永久保存，并且可随时查阅、编辑或共享。，对于团队协作，任务文件可提交至Git仓库，使团队能同步查看AI的工作内容、当前状态和历史决策。开发者无需在IDE与外部项目管理工具间频繁切换，也无需手动复制粘贴上下文，从而实现了真正为AI辅助编程优化的原生工作流。
