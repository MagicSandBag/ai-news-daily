Apideck CLI——一种比MCP消耗更少上下文资源的AI-agent接口，核心问题在于，当前流行的模型上下文协议（MCP）服务器在工具定义阶段会消耗大量上下文窗口（context window）令牌（tokens）。一个典型场景是，当连接GitHub、Slack和Sentry等服务（约40个工具）时，仅工具定义就可能消耗超过55,000个tokens，这占据了Claude模型200k上下文限制的四分之一以上，严重挤占了用于处理实际用户消息和任务执行的资源。每个MCP工具因其名称、描述、JSON schema、字段说明、枚举和系统指令，需要消耗550至1,400个tokens。对于拥有50多个端点的真实SaaS平台API，仅描述工具就可能耗尽50,000+ tokens。，为此，Apideck推出了Apideck CLI作为替代方案。它是一个为AI-agent设计的命令行接口，其核心创新在于大幅降低了上下文消耗。该方案用一个约80个tokens的简洁agent提示（prompt）替代了原本需要数万tokens的完整模式（schema）描述。其工作原理是“渐进式披露”（progressive disclosure）：AI-agent首先看到一个精简的命令概览，当需要执行具体操作时，可以通过运行带有`--help`参数的CLI命令来动态获取更详细的结构化信息和参数选项。这种设计将详细的结构化安全约束内置于CLI二进制文件中，而非通过冗长的文本描述传递给LLM。，Apideck CLI的主要优势与价值包括：极低的上下文占用，为实际任务对话留出充足空间；广泛的兼容性，任何能够执行shell命令的AI-agent均可直接使用，无需额外的协议支持；内置的结构安全性保障。它本质上提供了一种更简单、更高效的方式，让AI-agent能够安全、灵活地调用复杂的API功能，而无需在上下文窗口中以高昂代价承载完整的API规范。
