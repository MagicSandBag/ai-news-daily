Hypura 是一款专为 Apple Silicon 设计的、具备存储层级感知能力的大语言模型（LLM）推理调度器。其核心目标是让用户能够在内存容量有限的 Mac 上运行远超物理内存大小的模型，而不会导致系统崩溃。，传统方法（如朴素的 llama.cpp）在尝试加载超过物理内存的模型时，操作系统会频繁进行内存交换，最终触发内存不足（OOM）终止进程。Hypura 通过智能地根据模型架构、访问模式和硬件能力，将模型张量动态放置于 GPU、统一内存（RAM）和 NVMe 存储这三个层级中，从而解决了这一问题。，其技术关键在于深度理解模型结构并优化数据调度：，1. 关键组件常驻 GPU：对每个 Token 都需访问的归一化层（Norms）和嵌入层（Embeddings），将其固定在 GPU 内存中以获得最快访问速度。，2. 高效处理混合专家模型（MoE）：针对如 Mixtral 8x7B 这类 MoE 模型，Hypura 利用其稀疏性。它通过路由拦截在评估回调中识别每个 Token 实际激活的专家（例如 8 个中的 2 个），并仅从 NVMe 加载所需专家的权重数据，减少了高达 75% 的 I/O。，3. 智能缓存与预取：引入神经元缓存来跨 Token 缓存已加载的专家切片，利用时间局部性实现了 99.5% 的缓存命中率。同时，通过协同激活跟踪来预测下一个可能激活的专家，进行推测性预取。，4. 动态流式处理稠密层：对于占模型体积约 60% 的稠密前馈网络（FFN）权重（gate, up, down），Hypura 让其从 NVMe 流式传输，通过一个动态大小的池缓冲区进行加载，而注意力机制等计算则持续在 GPU 上进行。预取前瞻深度会根据可用内存自动调整。，实际效果显著：在 32GB 内存的 Mac Mini 上，Hypura 可以运行 31GB 的 Mixtral 8x7B 模型，速度达到 2.2 tok/s；运行 40GB 的 Llama 70B 模型，速度为 0.3 tok/s。而对于本就能完全装入内存的模型，Hypura 则能实现完整的 Metal GPU 加速，且无额外开销。，工作原理上，Hypura 读取 GGUF 模型文件，分析硬件配置（GPU 工作集大小、RAM、NVMe 带宽），并通过求解一个放置优化问题，为每个张量分配最优的存储层级（GPU、RAM 或 NVMe），从而实现超大模型在消费级 Apple 硬件上的可行推理。
