近日，GitHub上开源了一个名为“nah”的项目，它是一个为Claude Code设计的上下文感知权限防护系统。该项目旨在解决Claude Code现有权限系统仅提供“允许或拒绝”这种粗粒度控制的问题，通过引入更精细、智能的权限管理，让用户能够真正掌控AI助手对系统工具和文件的访问。，核心问题是，Claude Code当前的权限机制是按工具（如Bash、Read、Write）进行全局允许或拒绝，这在实际使用中缺乏灵活性且难以扩展。例如，执行`git push`通常是安全的，但`git push --force`可能具有破坏性；删除缓存文件`__pycache__`可以接受，但删除关键配置文件`~/.bashrc`则非常危险。即使精心配置权限列表，高智能的模型（如Claude Opus）也可能找到绕过方法，单纯维护一个拒绝列表效果有限。，“nah”系统通过一个PreToolUse钩子，在每次工具调用执行前进行实时拦截和评估。其核心功能是根据上下文规则对工具调用进行毫秒级分类和检查：，- 对于Bash命令，它会进行结构分类，分析操作类型、管道组合并进行Shell解包。，- 对于Read操作，它会检测敏感路径（如`~/.ssh`、`~/.aws`、`.env`等）。，- 对于Write和Edit操作，它会检查路径、项目边界，并深入检查内容本身，以识别潜在的密钥泄露、数据外泄或破坏性负载。，对于模糊或难以判断的情况，系统可选地将决策路由给一个LLM进行辅助判断。所有决策都会被记录并可追溯，确保了透明性。，该项目易于安装（通过`pip install nah`后运行`nah install`），并提供了开箱即用的默认配置，同时也支持用户自定义规则。项目文档中强调，用户应避免使用Claude Code自带的`--dangerously-skip-permissions`选项，因为在绕过模式下，命令会在“nah”拦截之前就异步执行，失去保护作用。，为了帮助用户理解其防护能力，项目提供了一个安全演示（在Claude Code中运行`/nah-demo`），该演示会引导用户体验25个涉及8种威胁类别（包括远程代码执行、数据窃取、混淆命令等）的实时案例，整个过程约需5分钟。，总之，“nah”项目为Claude Code用户提供了一个可控、上下文感知且可审计的权限防护层，在提供便利的同时，显著增强了使用AI编程助手时的系统安全性和可控性。
