Anthropic发布了Claude Opus 4.7模型，其官方迁移指南指出，新分词器（Tokenizer）产生的token数量约为4.6版本的1.0至1.35倍。然而，通过实际测量发现，在技术文档上token数量增加了1.47倍，在一个真实的CLAUDE.md文件上增加了1.45倍。这意味着大多数Claude Code用户的实际内容消耗更接近官方范围的上限，而非中间值。，尽管模型定价和配额保持不变，但新分词器导致每个提示消耗更多token，从而带来一系列隐性成本：最大上下文窗口消耗更快，缓存前缀每轮对话成本更高，速率限制会更早触发。这促使人们探究Anthropic做出这一权衡的原因及其价值。，为量化成本，作者使用Anthropic提供的免费无推理token计数API（POST /v1/messages/count_tokens），对同一内容在4.6和4.7模型上分别进行测量。实验分为两部分：第一部分使用七份来自真实Claude Code用户的样本，包括CLAUDE.md文件、用户提示、博客文章、git日志、终端输出、堆栈跟踪和代码差异；第二部分使用十二份合成样本，涵盖英文散文、代码、结构化数据、中日韩文字、表情符号和数学符号等类型，以观察不同内容类型的比率差异。，真实Claude Code内容测量结果显示，加权平均比率为1.325倍（从8,254个token增至10,937个token）。其中，CLAUDE.md文件的比率最高（1.445倍），代码差异的比率最低（1.212倍）。在合成样本中，技术文档的比率最高（1.47倍），而中文和日文散文的比率几乎为1.01倍，变化最小。对于英文和代码内容子集，token增长普遍显著。，核心结论是：Claude 4.7的新分词器在多数实际内容（尤其是技术性内容）上带来的token增长超出官方预估，直接导致使用成本上升。用户需关注其上下文窗口、缓存成本和速率限制受到的影响。Anthropic可能通过改进分词效率或模型性能来平衡这一变化，但其实际收益仍需进一步评估。

DeepGEMM是由深度求索（deepseek-ai）开源的一个统一、高性能的张量核心（Tensor Core）内核库。它将现代大型语言模型（LLM）的关键计算原语集成到一个简洁、统一的CUDA代码库中。核心功能包括支持多种精度的通用矩阵乘法（GEMM），如FP8、FP4和BF16；集成了具有重叠通信功能的融合混合专家模型（fused MoE with overlapped communication，即Mega MoE）；为闪电索引器（lightning indexer）服务的多头查询注意力（MQA）评分内核；以及超连接（HyperConnection, HC）等。，该库的设计追求简洁与高效。它借鉴了CUTLASS和CuTe的一些概念，但避免重度依赖其模板或代数系统，核心内核函数数量有限，使其成为一个清晰、易于学习NVIDIA GPU内核优化技术的资源。所有内核均通过一个轻量级的即时（JIT）编译模块在运行时编译，安装时无需进行CUDA编译，简化了部署流程。尽管设计轻量，但其性能在各种矩阵形状下均能达到或超过专家调优的库。，近期的重要更新包括：支持用于DeepSeek v3.2闪电索引器的评分内核；全面支持SM90/SM100架构并进行了代码重构，采用了低CPU开销的JIT CPP模块；提供了密集层和MoE层的权重梯度内核以支持反向传播；以及通过启用NVRTC实现了高达10倍的编译加速（可通过设置DG_JIT_USE_NVRTC=1启用，但在某些情况下可能会有性能损失）。据报告，在H800 GPU上，其性能最高可达1550 TFLOPS。项目持续演进，最新动态包括对Mega MoE、FP8xFP4 GEMM、FP4索引器等的支持。
